## 1. 개요

### 1.1 문서 목적

이 문서는 Spring PetClinic 프로젝트의 **모든 화면 UI/UX 규칙과 구조**를 하나의 문서로 통합하여:
- ✅ 개발자 간 일관된 UI/UX 구현
- ✅ 신규 기능 추가 시 빠른 참조
- ✅ 유지보수 및 협업 효율성 향상
- ✅ 사용자 경험 일관성 확보

### 1.2 핵심 설계 원칙

```
✅ 일관성 (Consistency)
   - 동일한 요소는 동일한 스타일
   - 모든 게시판은 동일한 레이아웃 패턴

✅ 직관성 (Intuitiveness)
   - 사용자가 설명 없이도 이해 가능
   - 아이콘 + 텍스트 조합

✅ 피드백 (Feedback)
   - 사용자 액션에 즉각적인 반응
   - Toast 알림, 모달 확인

✅ 접근성 (Accessibility)
   - 모바일 포함 다양한 환경 지원
   - 반응형 디자인 필수

✅ 오프라인 우선 (Offline First)
   - CDN 사용 금지
   - 모든 라이브러리 로컬 내장
```

### 1.3 기술 스택

| 구분 | 기술 | 버전 | 용도 |
|------|------|------|------|
| **템플릿 엔진** | Thymeleaf | 3.1.3 | 서버 사이드 렌더링 |
| **CSS 프레임워크** | Bootstrap | 5.3.x | 반응형 레이아웃 |
| **아이콘** | Bootstrap Icons | 1.11.x | UI 아이콘 |
| **아이콘** | Font Awesome | 6.x | 추가 아이콘 |
| **에디터** | Quill | 2.0.x | 리치 텍스트 에디터 (로컬 내장) |
| **스크립트** | JavaScript | ES6+ | 클라이언트 동작 |

## 2. 레이아웃 시스템 (Layout)

### 2.1 전체 레이아웃 구조

```
┌─────────────────────────────────────────────┐
│              Header (상단 헤더)               │
│  - 로그인/회원가입 (비로그인 시)              │
│  - 닉네임/마이페이지/로그아웃 (로그인 시)     │
│  - 관리자 전용: 설정 링크                    │
├─────────────────────────────────────────────┤
│            Navigation Bar (네비)             │
│  - 솔 동물의료센터 소개                      │
│  - 일반클리닉                                │
│  - 특화클리닉                                │
│  - 건강검진프로그램                          │
│  - 최소침습시술                              │
│  - 온라인상담 (온라인상담, 자주묻는질문)     │
│  - 커뮤니티 (공지사항, 고객포토)             │
├─────────────────────────────────────────────┤
│                                               │
│              Content Area (본문)             │
│                                               │
│                                               │
├─────────────────────────────────────────────┤
│              Footer (하단 푸터)               │
│  - 사이트맵                                   │
│  - 개인정보취급방침                          │
│  - 서비스이용약관                            │
└─────────────────────────────────────────────┘
```

### 2.2 Header 구성

**파일**: `fragments/layout.html`

#### 2.2.1 비로그인 상태
```html
<a href="/">HOME</a> | <a href="/login">로그인</a> | <a href="/register">회원가입</a>
```

#### 2.2.2 로그인 상태
```html
[닉네임]님 | <a href="/mypage">마이페이지</a> | [관리자 전용: 설정] | 로그아웃
```

#### 2.2.3 관리자 전용 메뉴
```html
<i class="bi bi-gear"></i> 설정
```
- **권한**: `hasRole('ADMIN')`
- **링크**: `/admin/settings`
- **색상**: `text-danger` (빨간색으로 구분)

#### 2.2.4 스타일 규칙
| 요소 | 속성 | 값 |
|------|------|-----|
| 폰트 크기 | `font-size` | `0.95rem` |
| 링크 간격 | `padding-left/right` | `0.5rem` (px-2) |
| 구분선 간격 | `padding-left/right` | `0.25rem` (px-1) |
| 링크 색상 | `color` | `link-secondary` (회색) |
| 관리자 색상 | `color` | `link-danger` (빨간색) |

## 3. 로그인/인증 화면

### 3.1 로그인 페이지 (`user/login.html`)

#### 3.1.1 화면 구성
```
┌──────────────────────────────────┐
│          로그인 (제목)            │
├──────────────────────────────────┤
│  📧 아이디 *                      │
│  [입력창]                         │
├──────────────────────────────────┤
│  🔒 비밀번호 *                    │
│  [입력창]                         │
├──────────────────────────────────┤
│  ☑ 아이디 저장                    │
│  ☑ 자동 로그인                    │
├──────────────────────────────────┤
│  [로그인 버튼 - 전체 너비]        │
├──────────────────────────────────┤
│  비밀번호 찾기 | 회원가입         │
└──────────────────────────────────┘
```

#### 3.1.2 UI 규칙
- **카드 크기**: 최대 `500px`
- **여백**: `p-5` (카드 내부)
- **버튼**: 전체 너비 (`width: 100%`), 높이 `42px`
- **링크**: 하단 중앙 정렬, 구분선 `|`

#### 3.1.3 새로운 기능 (2025-11-26 추가)

**ID 저장 기능**:
```html
<input type="checkbox" id="save-username" name="save-username">
<label for="save-username">
  <i class="bi bi-bookmark"></i> 아이디 저장
</label>
```
- ✅ 체크 시 아이디 30일간 저장 (쿠키)
- ✅ 다음 방문 시 자동으로 아이디 입력
- ✅ JavaScript 구현 (페이지 로드/폼 제출 이벤트)

**자동 로그인 (Remember-Me)**:
```html
<input type="checkbox" id="remember-me" name="remember-me">
<label for="remember-me">
  <i class="bi bi-shield-check"></i> 자동 로그인 (7일간 유지)
</label>
```
- ✅ Spring Security Remember-Me 기능
- ✅ 7일간 로그인 상태 유지
- ✅ 토큰 기반 인증

#### 3.1.4 검증 규칙
- **아이디**: 필수 입력, 4자 이상
- **비밀번호**: 필수 입력, 8자 이상
- **ID 저장**: 선택 사항, 쿠키 저장 (30일)
- **자동 로그인**: 선택 사항, 토큰 저장 (7일)
- **실패 시**: Flash 메시지 표시

### 3.2 회원가입 페이지 (`user/register.html`)

#### 3.2.1 입력 필드
| 필드 | 필수 여부 | 검증 규칙 |
|------|----------|----------|
| 아이디 | ✅ 필수 | 4~20자, 영문+숫자 |
| 비밀번호 | ✅ 필수 | 8자 이상, 영문+숫자+특수문자 |
| 비밀번호 확인 | ✅ 필수 | 비밀번호와 일치 |
| 이름 | ✅ 필수 | 2~10자, 한글 또는 영문 |
| 닉네임 | ✅ 필수 | 2~20자, 게시판에 표시 |
| 이메일 | ✅ 필수 | 이메일 형식 검증 |
| 휴대폰 | ✅ 필수 | 자동 포맷 (010-0000-0000) |

#### 3.2.2 휴대폰 자동 포맷
```javascript
// 숫자만 입력해도 010-0000-0000 형식으로 자동 변환
document.getElementById('phone').addEventListener('input', function(e) {
  let value = e.target.value.replace(/\D/g, ''); // 숫자만 추출
  
  if (value.length <= 3) {
    e.target.value = value;
  } else if (value.length <= 7) {
    e.target.value = value.slice(0, 3) + '-' + value.slice(3);
  } else {
    e.target.value = value.slice(0, 3) + '-' + value.slice(3, 7) + '-' + value.slice(7, 11);
  }
});
```

| 유형 | `min-width` | 예시 |

| 색상 | Bootstrap 클래스 | 용도 |
