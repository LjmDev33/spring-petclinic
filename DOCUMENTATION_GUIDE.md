# 📚 문서 관리 가이드

## 📋 개요

이 프로젝트는 **체계적인 문서 관리 시스템**을 운영합니다.  
모든 변경사항은 자동으로 관련 문서에 반영되며, 이력이 기록됩니다.

---

## 📂 문서 구조

```
spring-petclinic/
├── CHANGELOG.md                    # 변경 이력 (메인)
├── PROJECT_DOCUMENTATION.md        # 상세 문서 (40+ KB)
├── QUICK_REFERENCE.md              # 빠른 참조 (8+ KB)
├── DOCUMENTATION_GUIDE.md          # 이 파일 - 문서 관리 가이드
└── README.md                       # 프로젝트 소개
```

### 각 문서의 역할

| 문서 | 크기 | 용도 | 업데이트 빈도 |
|------|------|------|--------------|
| **CHANGELOG.md** | 8+ KB | 모든 변경 이력 기록 | 변경 시마다 |
| **PROJECT_DOCUMENTATION.md** | 40+ KB | 전체 아키텍처, 설계, 개발 가이드 | 구조 변경 시 |
| **QUICK_REFERENCE.md** | 8+ KB | 핵심 요약, 코드 예시 | 주요 변경 시 |
| **DOCUMENTATION_GUIDE.md** | 3 KB | 문서 관리 방법 | 규칙 변경 시 |
| **README.md** | 가변 | 프로젝트 소개 | 배포 전 |

## 🔄 문서 업데이트 프로세스

### 변경사항 발생 시

```
1. 소스코드 수정
   ↓
2. CHANGELOG.md 기록 (변경 이력)
   ↓
3. PROJECT_DOCUMENTATION.md 업데이트 (상세 내용)
   ↓
4. QUICK_REFERENCE.md 업데이트 (핵심 요약)
   ↓
5. 소스코드 주석 업데이트 (JavaDoc)
   ↓
6. Git 커밋 & 푸시
```

## 🚀 Git 커밋 메시지 규칙

### 커밋 메시지 포맷
```
<타입>(<범위>): <제목>

<본문>

<푸터>
```

### 타입
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅
- `refactor`: 코드 리팩토링
- `test`: 테스트 코드
- `chore`: 빌드 설정

### 예시
```
feat(counsel): 댓글 수정 기능 추가

- PUT /counsel/detail/{postId}/comments/{commentId} API 구현
- 비밀번호 검증 로직 추가
- 관련 문서 업데이트

Resolves #123
```
